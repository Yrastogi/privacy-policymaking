Privacy Policy for Calculator Vault
🔒 Local-Only Privacy

Effective date:

App package name: com.yash.securevault
Store listing: Calculator Vault – Secret Photo Locker

This privacy policy applies to Calculator Vault (internal Expo app name: Calculator), a secure media vault app built with Expo and React Native. The app is designed to store user-selected photos and videos locally on the device.

1. No Data Collection – Offline First

Calculator Vault does NOT collect, transmit, or share any personal data. There are no analytics SDKs, no crash reporting services, and no third-party tracking libraries in the current app code.

2. What Information Is Stored Locally

  • Encrypted PIN hash + salt – The app stores the user PIN as a SHA-256 hash with a random salt in secure storage via expo-secure-store. The raw PIN is never stored in plaintext.
  • Master encryption key – A randomly generated vault key is stored securely with expo-secure-store and is used to encrypt all vault data.
  • Encrypted media files – Imported photos and videos are encrypted using AES-256-CBC with a random IV and saved in the app’s private documents folder under .vault/.
  • Encrypted metadata – File metadata (including original filename, MIME type, extension, and item state) is encrypted and saved inside .vault/metadata.json. Metadata is not stored in plaintext.

All data remains on the user’s device. No app data is uploaded to any cloud service or external server.

3. Permissions Used & Why

  • READ_MEDIA_IMAGES / READ_MEDIA_VIDEO – Used only when the user explicitly selects photos or videos to import into the vault.
  • expo-image-picker is used for gallery import. The app does not automatically scan your library.
  • VIBRATE – Used for haptic feedback in the app interface.
  • RECORD_AUDIO – Present in the Android manifest but not actively used by the current release of the app.
  • USE_BIOMETRIC / USE_FINGERPRINT – Included in Android permissions configuration but the current app code does not implement biometric unlock in the active release.
  • No internet permission is requested for normal use, and the app does not require network access to secure or view vault contents.

4. How Your Media Is Protected

  • AES-256-CBC encryption – Each file is encrypted with a random IV and the app’s master key.
  • Obfuscated storage names – Vault files are saved under randomized .dat names so nobody can identify them by browsing the files.
  • Temporary cache handling – Decrypted content is written only to the app cache directory for preview/export and is removed when the app closes the viewer or deletes the item.
  • App-private storage – Encrypted vault data is stored in the app’s private filesystem. Android/iOS remove this data automatically when the app is uninstalled.

5. Data Deletion

Vault data is deleted when:

  • You delete an item inside the vault – the encrypted file and its metadata entry are removed.
  • You uninstall the app – the operating system deletes all app-private storage, including .vault/ and cache directories.

If the user enables the optional "delete original after import" setting, the app will also attempt to remove the selected source media from the device gallery after import.

6. Third-Party Services

None. The current app code does not integrate external analytics, advertising, crash reporting, or cloud services. Dependencies such as React Native, Expo modules, and crypto-js are used only for local app functionality.

7. Children’s Privacy

Calculator Vault does not collect any personal information from children under 13. The app is not designed for kids, and because it stores data locally, no child data is transmitted to third parties.

8. Changes to This Privacy Policy

We may update this policy to reflect code or configuration changes. Any updates will be posted on this page and the effective date will be updated accordingly.
